## Why this goal and not another

After 22 phases of Stockerly, the owner (Adrian) admitted on 2026-05-14 that "the project drifted a lot", that "I lost the thread on a redesign we didn't finish", and that "I'm not sure if it solves my original needs". The project had 2080 specs, ~94% coverage, green CI — respectable code — but its working method was ad-hoc ("Phase 23 — TBD"), the docs were aspirational fiction (a 439-line PRD for 3 personas when the real user is one), and the visual redesign was abandoned mid-way.

This sprint **doesn't implement a single product feature.** It implements the project's operating system: vision, anti-patterns, expert panel, sprint protocol, GitHub board, code audit, seed backlog.

The hypothesis: a sprint dedicated to method saves the next 6 months of drift. The rest of the roadmap (S2-S7) is built on this foundation.
